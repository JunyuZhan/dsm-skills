# 常见故障排除 (Troubleshooting)

NAS 遇到问题不要慌，这里有 10 个常见故障的自救指南。

## 1. 忘记管理员密码
- **症状**：无法登录 DSM。
- **解决方法**：
    1.  找到 NAS 背面的 **RESET** 按钮。
    2.  用回形针按住约 **4 秒**，直到听到一声“哔”。
    3.  立即松开。这会重置 `admin` 密码为空，并恢复网络设置（IP 变回 DHCP）。**数据不会丢失**。
    4.  注意：如果按住超过 10 秒（听到三声哔），系统会重装 DSM！千万别按久了。

## 2. 无法访问 NAS (网络问题)
- **症状**：Synology Assistant 搜不到，网页打不开。
- **排查**：
    - 检查网线指示灯是否闪烁。
    - 检查路由器 DHCP 列表，确认 NAS 是否获取到了 IP。
    - 尝试直接用网线连接电脑和 NAS，将电脑 IP 设为 `169.254.1.1`，尝试访问 NAS。

## 3. 硬盘无法休眠
- **症状**：硬盘灯一直闪，噪音大，费电。
- **原因**：
    - 后台有服务在读写（索引、缩略图生成、Cloud Sync 同步）。
    - 开启了 QuickConnect 或 DDNS 心跳检测。
    - 安装了 Docker 容器（这是最常见原因，容器通常会持续写日志）。
- **排查**：开启“休眠日志”查看唤醒源，或拔掉网线测试是否休眠。

## 4. SMB 传输速度慢
- **症状**：千兆局域网只有 10MB/s。
- **排查**：
    - 检查网线是否为 Cat5e 或 Cat6（千兆线）。
    - 检查交换机/路由器端口是否协商在 1000Mbps。
    - 检查电脑网卡驱动。
    - 尝试关闭 SMB 加密（在文件服务 > SMB > 高级设置中）。

## 5. Docker 镜像下载失败
- **症状**：`Registry` 查询超时，下载速度 0。
- **原因**：Docker Hub 被墙。
- **解决**：配置镜像加速器（如阿里云、网易云等国内镜像源）。或者使用代理服务器（在 Docker 设置中配置 HTTP 代理）。

## 6. 存储空间损毁 (Crashed)
- **症状**：存储池状态变为“红色”，只能读不能写。
- **原因**：硬盘坏道过多，或 RAID 阵列丢失数据。
- **紧急措施**：
    1.  **立刻停止一切写入操作！**
    2.  尽快通过 File Station 或 SMB 将重要数据拷贝出来。
    3.  如果是硬盘故障，更换硬盘并修复存储池。
    4.  如果是文件系统错误，可能需要联系群晖技术支持。

## 7. CPU 占用率 100%
- **症状**：界面卡顿，响应极慢。
- **排查**：
    - 打开“资源监控” > “进程”。
    - 常见祸首：`synoelasticd` (索引), `convert` (生成缩略图), `synology-drive` (同步)。
    - 解决：暂停索引服务，或等待后台任务完成。

## 8. 小文件传输极慢
- **症状**：拷贝几万个几 KB 的文件，速度只有几百 KB/s。
- **原因**：这是 SMB 协议和机械硬盘的物理特性决定的（IOPS 瓶颈）。
- **解决**：先打包成一个压缩包 (zip/rar)，再传输，速度会快得多。

## 9. Web Station 403/404 错误
- **症状**：搭建的网站无法访问。
- **原因**：权限问题。
- **解决**：确保网站根目录及其父目录赋予了 `http` 用户组（或 `System Internal User > http`）的读取权限。

## 10. 系统升级后套件无法运行
- **症状**：升级 DSM 7.x 后，部分第三方套件显示“不兼容”。
- **解决**：
    - 在套件中心点击“修复”。
    - 如果没有修复按钮，需卸载后重新安装最新版（注意备份配置）。
    - 对于不再支持的套件（如旧版 Docker），需寻找替代品或使用 Docker 部署。
