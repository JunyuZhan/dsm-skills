# 黑群晖安装指南 (RR 引导)

对于非群晖官方硬件（DIY NAS、旧电脑、虚拟机），我们可以通过“黑群晖”方式安装 DSM 系统。目前最推荐、最简单的方式是使用 **RR (Redpill Recovery)** 引导。

!!! warning "免责声明"
    黑群晖仅供个人学习研究使用，请勿用于生产环境或商业用途。为了数据安全和稳定性，建议购买群晖官方硬件（白群晖）。

## 什么是 RR 引导？

RR (Redpill Recovery) 是基于 RedPill 项目开发的增强版引导程序。相比传统的引导方式，它的优势在于：
*   **Web 界面配置**：无需修改代码，直接在浏览器中选择型号、版本、插件。
*   **自动驱动**：自动检测硬件并加载网卡、HBA 卡驱动。
*   **在线更新**：引导程序本身支持在线 OTA 更新。
*   **插件系统**：一键开启人脸识别补丁、NVMe 存储池支持等。

## 1. 准备工作

*   **硬件**：一台 x86 架构的电脑（Intel/AMD 均可）。
    *   内存建议 ≥ 4GB。
    *   至少一块 SATA 硬盘（用于安装系统）。
*   **U 盘**：一个 2GB 以上的 U 盘，用于制作引导盘（安装后需一直插在机器上）。
*   **软件**：
    *   [RR 引导镜像](https://github.com/RROrg/rr/releases) (下载 `rr-x.x.x.img.zip`)。
    *   [Rufus](https://rufus.ie/) (写盘工具)。
    *   [Synology Assistant](https://www.synology.cn/zh-cn/support/download) (群晖助手，用于查找 IP)。

## 2. 制作引导盘

1.  将 U 盘插入电脑。
2.  打开 **Rufus**。
3.  **设备**：选择你的 U 盘。
4.  **引导类型选择**：选择下载好的 `rr-x.x.x.img` 文件。
5.  点击 **开始**，等待写入完成。

## 3. 首次启动与编译

1.  将 U 盘插入 NAS，连接网线、键盘、显示器。
2.  开机，进入 BIOS 设置 **U 盘为第一启动项**。
3.  保存重启，屏幕会出现 RR 的启动菜单，默认会自动进入。
4.  等待屏幕跑码结束，最后会显示一个 IP 地址和 URL（通常是 `http://IP:7681`）。
5.  **在同一局域网的电脑浏览器中**，访问该 URL。

## 4. RR 配置向导

进入网页版配置界面后：

### A. 选择语言
使用方向键选择 `zh_CN` (简体中文)。

### B. 选择型号 (Choose a model)
这是最关键的一步。推荐选择：

| 型号 | 架构 | 推荐理由 | 缺点 |
| :--- | :--- | :--- | :--- |
| **DS920+** | Geminilake (J4125) | **家用首选**。支持 Intel 核显硬件解码 (J4125/N5105 等)，支持 NVMe 缓存。 | 对 AMD CPU 兼容性一般。 |
| **SA6400** | EPYC 7002 (Linux 5.10) | **兼容性之王**。内核新，支持 12 代+ Intel CPU (大小核)、AMD CPU、以及各种新网卡/阵列卡。支持超多盘位。 | 默认不支持核显转码 (需折腾)。 |
| **DS3622xs+** | Broadwellnk | **企业级稳**。适合不需要核显转码、追求极致稳定的场景。 | 系统限制较多，无可玩性插件少。 |
| **DVA1622** | Geminilake | **监控专用**。自带 8 个免费摄像头授权 (其他机型只有 2 个)。 | AI 功能需特定显卡。 |

*   *一般建议：要硬解选 DS920+，不要硬解或硬件太新选 SA6400。*

### C. 选择版本 (Choose a version)
选择最新的 DSM 7.2.x 版本即可。

### D. 插件与高级设置 (Addons)
在 `Addons` 菜单中，可以开启一些实用功能：
*   `codecpatch`: 解锁 HEVC/AAC 解码（配合 Video Station）。
*   `nvmestorage`: 允许在 NVMe SSD 上创建存储池。
*   `cpuinfo`: 在控制面板显示真实的 CPU 型号。

### E. 编译引导 (Build the loader)
配置完成后，选择 **Build the loader**。系统会自动下载所需文件并编译。完成后会自动跳转到“启动”菜单，选择 **Boot**。

## 5. 安装 DSM 系统

1.  NAS 重启后，等待几分钟。
2.  在电脑浏览器访问 `http://NAS_IP:5000`，或者使用 **Synology Assistant** 搜索。
3.  会出现“欢迎”界面，点击 **安装**。
4.  **上传 .pat 文件**：
    *   RR 引导通常会自动下载好对应的 `.pat` 文件。
    *   如果没有，去 [群晖官网下载中心](https://www.synology.cn/zh-cn/support/download) 下载对应型号和版本的系统包手动上传。
5.  **注意**：安装过程中会提示“删除硬盘所有数据”，确认即可。
6.  等待安装完成（约 10 分钟），系统会自动重启。

## 6. 洗白 (SN/MAC)

“洗白”是指修改引导中的序列号 (SN) 和 MAC 地址，使其伪装成正版机器，以使用 QuickConnect 和转码功能。

*   **全白**：正版 SN + 对应 MAC。可使用 QC、转码、缩略图。
*   **半白**：算号器生成的 SN + 真实 MAC。**不能用 QC**，但可以使用相册缩略图、人脸识别、Video Station 转码。
*   **黑**：默认状态。部分功能受限。

**在 RR 中修改 SN/MAC**：
1.  重启 NAS，在引导菜单选择 `Configure loader` (或浏览器访问 `IP:7681`)。
2.  进入 `Cmdline` 菜单。
3.  修改 `sn` 和 `mac1` 值。
4.  重新编译引导并重启。

## 7. 常见问题

*   **找不到 IP**：
    *   检查网线是否插好。
    *   网卡驱动未加载：尝试换一个型号 (如 SA6400 兼容性更好)。
*   **卡在 55% 安装失败**：
    *   通常是 U 盘 VID/PID 错误（RR 会自动获取，一般没事）。
    *   或者硬盘有坏道/分区表混乱，建议在 PE 下格式化硬盘再试。
*   **无限还原/循环安装**：
    *   引导盘 VID/PID 没对上。
    *   SATA 端口映射错误（SataPortMap）。RR 通常自动处理，若不行需手动调整。

!!! tip "后续升级"
    使用 RR 引导的黑群晖，通常可以直接在 DSM 控制面板中进行**小版本更新** (Update x)。但**跨大版本更新** (如 7.1 -> 7.2) 必须先更新 RR 引导本身，否则会无法开机。
