# Cloud Sync 深度指南：公有云与私有云的桥梁

很多人买 NAS 是为了摆脱百度网盘的限速和审查，但成年人的世界不做选择题，我们**全都要**。Cloud Sync 就是连接你的 NAS 和公有云（百度网盘、OneDrive、Google Drive、阿里云 OSS 等）的桥梁。

## 1. 为什么需要 Cloud Sync？

*   **异地容灾 (3-2-1 备份)**：NAS 放在家里，万一火灾或硬盘全挂了怎么办？Cloud Sync 可以实时把重要数据（照片、文档）加密上传到 OneDrive 或百度网盘。
*   **数据交换**：在公司用 OneDrive 改了文档，自动同步到家里的 NAS；回家打开 NAS 直接接着写。
*   **利用公有云带宽**：把热门电影同步到百度网盘，分享给朋友下载，利用百度的带宽而不是你家可怜的上行带宽。

## 2. 同步模式详解 (Sync Direction)

这是 Cloud Sync 最核心的设置，选错了会导致数据丢失或混乱。

*   **双向同步 (Bidirectional)**：
    *   **逻辑**：NAS 改了，云端跟着改；云端改了，NAS 跟着改。
    *   **场景**：协同办公。OneDrive 里的工作文档。
    *   **注意**：如果云端中毒了，NAS 也会同步中毒。
*   **仅下载 (Download only)**：
    *   **逻辑**：云端有新文件，下载到 NAS。NAS 删了文件，云端不删（可选）。
    *   **场景**：把百度网盘里的资源下载到 NAS 看。或者把公司 Google Drive 的共享资料备份回来。
*   **仅上传 (Upload only)**：
    *   **逻辑**：NAS 有新文件，上传到云端。
    *   **场景**：**备份**。把 NAS 里的照片备份到百度网盘。
    *   **勾选建议**：建议勾选“当文件在本地被删除时，不要在公有云上删除”。这样即使你误删了 NAS 文件，云端还有一份“后悔药”。

## 3. 数据加密 (Data Encryption) —— 隐私保护伞

你不敢把私密照片传到百度网盘，怕被审查或泄露？Cloud Sync 的**客户端加密**功能完美解决这个问题。

*   **原理**：数据在离开 NAS 之前，先用 AES-256 密码加密，变成一堆乱码文件，然后才上传到网盘。
*   **设置**：
    *   在创建任务时，勾选 **数据加密**。
    *   设置一个密码。
    *   **导出密钥**：系统会让你下载一个 `key.zip` 或私钥文件。**务必保存好！**
*   **效果**：
    *   **在网盘网页上看**：文件名被加密（如 `a1b2c3d4...`），内容也是乱码。网盘审查机制无法识别。
    *   **在 NAS 上看**：文件是正常的。
*   **如何恢复**：
    *   如果 NAS 挂了，你从网盘下载回来的文件是加密的。
    *   你需要用 **Synology Cloud Sync Decryption Tool** (电脑端工具) 配合你的密码/密钥来解密。

## 4. 百度网盘的特殊限制

*   **同步文件夹**：百度限制第三方应用只能访问 `/apps/Cloud Sync` 文件夹。你**不能**同步百度网盘根目录下的所有文件。
    *   **技巧**：在百度网盘网页版里，把你想要同步的文件夹（如“我的资源”）**移动**或**复制**到 `/apps/Cloud Sync` 下面。
*   **速度限制**：
    *   **上传**：通常能跑满你的上行带宽。
    *   **下载**：如果你没有百度 SVIP，下载速度会非常慢（几十 KB/s）。Cloud Sync 救不了非会员的下载速度。

## 5. 阿里云对象存储 (OSS) / 腾讯云 COS

对于极客和中小企业，与其用网盘，不如用对象存储。

*   **优势**：
    *   **可靠性**：99.999999999% 的数据持久性。
    *   **速度**：不限速，取决于你的带宽。
    *   **成本**：归档存储（Archive）非常便宜（1TB 也就几十块钱一个月），适合存“万一要用”的冷数据。
*   **设置**：
    *   选择 **S3 Storage** (兼容协议) 或 **阿里云 OSS**。
    *   填入 `Access Key` 和 `Secret Key`。
    *   **存储桶 (Bucket)**：建议设置为“私有读写”。
    *   **存储类型**：在阿里云后台设置生命周期规则，把上传的文件自动转为“归档存储”或“冷归档”，大幅降低成本。

## 6. 排除策略 (Filter)

不要什么垃圾文件都同步。

*   **文件过滤器**：
    *   **文件名**：排除 `*.tmp`, `*.log`, `Desktop.ini`, `.DS_Store`。
    *   **文件夹**：排除 `node_modules`, `.git`, `@eaDir` (缩略图文件夹)。
    *   **大小**：排除大于 5GB 的文件（如果你只是想备份文档）。
*   **重要性**：如果不排除 `node_modules`，包含几万个小文件，同步任务会卡死，云端 API 请求次数也会爆炸（对象存储是按请求次数收费的）。

## 7. 常见问题 (FAQ)

*   **Q: 为什么 Cloud Sync 总是显示“同步中”但没动静？**
    *   A: 可能是卡在某个被占用的文件上，或者文件名包含非法字符（如 `/ \ : * ? " < > |`）。检查日志。
*   **Q: Cloud Sync 能替代 Hyper Backup 吗？**
    *   A: **不能**。Cloud Sync 是“同步”（文件级），Hyper Backup 是“备份”（块级）。
    *   **区别**：如果你在 NAS 删了一个文件，Cloud Sync (双向模式) 会立马把云端也删了。而 Hyper Backup 会保留历史版本，能救命。
    *   **建议**：重要数据用 Hyper Backup；协同办公/非关键数据用 Cloud Sync。
