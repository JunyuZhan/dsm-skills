# File Station 深度指南：从入门到精通

File Station 是 DSM 的文件管理器，但它比 Windows 资源管理器强大得多。掌握以下技巧，你的文件管理效率将提升 10 倍。

## 1. 键盘快捷键 (Keyboard Shortcuts)

像操作本地电脑一样操作 NAS。

*   **F2**：重命名文件。
*   **Del**：删除文件。
*   **Ctrl + C / Ctrl + V**：复制/粘贴（仅限 File Station 内部，不能跨电脑）。
*   **Ctrl + X**：剪切。
*   **Ctrl + A**：全选。
*   **Ctrl + F**：查找。
*   **Shift + 拖拽**：**移动**文件（默认拖拽是复制）。
*   **Ctrl + 拖拽**：**复制**文件。
*   **Alt + 拖拽**：在同一浏览器窗口内，把电脑桌面的文件拖进 File Station 是上传；**反之不行**（浏览器限制）。

## 2. 后台任务管理 (Background Tasks)

当你复制 1TB 的数据时，不需要傻傻地盯着进度条，也不用担心关掉浏览器会中断。

*   **机制**：File Station 的所有操作都是在 NAS 后台运行的。
*   **操作**：
    1.  开始复制/移动/解压。
    2.  点击右上角的 **旋转图标 (后台任务)**。
    3.  你可以看到进度、速度、剩余时间。
    4.  **技巧**：你可以直接**关闭浏览器**甚至**关电脑**。任务会在 NAS 后台继续运行。下次登录时再查看结果。

## 3. 高级搜索语法 (Search Operators)

找不到文件？搜索框支持类似 Google 的语法。

*   **扩展名搜索**：`*.jpg` 或 `extension:jpg`
*   **时间搜索**：`date:2023-01-01..2023-12-31` (搜索去年的文件)
*   **大小搜索**：`size:>1GB` (找大文件清理空间)
*   **所有者搜索**：`owner:admin`
*   **组合搜索**：`extension:mkv size:>5GB` (找高清电影)

## 4. 远程连接 (Remote Connection)

把公有云和其他 NAS 挂载进来，统一管理。

*   **挂载云盘**：工具 > 远程连接 > 连接设置。
    *   **支持**：Google Drive, OneDrive, Dropbox, Box, Baidu Cloud (部分版本)。
    *   **限制**：
        *   **API 限制**：公有云通常有 API 调用频率限制，如果在此处大量传输小文件（如 node_modules），可能会被暂时封禁 API。
        *   **单文件大小**：部分协议（如 WebDAV）对单文件大小有限制。
*   **挂载 SMB/NFS**：工具 > 装载远程文件夹 > CIFS 共享文件夹。
    *   **场景**：把老 NAS 的电影文件夹挂载到新 NAS 上，直接拖拽复制，利用两台 NAS 间的千兆/万兆局域网传输，不经过电脑，速度极快。
    *   **排坑**：
        *   如果连接失败，检查目标 NAS 的 **SMB 版本**（控制面板 > 文件服务 > SMB > 高级设置）。通常设为 SMB2 或 SMB3。
        *   确保目标文件夹权限允许当前用户访问。
*   **FTP/SFTP/WebDAV**：
    *   File Station 本身也可以作为客户端连接 FTP/SFTP 服务器。
    *   **工具 > 远程连接 > 连接设置** 中选择 FTP/SFTP/WebDAV。
    *   这对于从 VPS 或其他服务器拉取备份文件非常有用。

## 5. 文件请求与分享 (File Request & Sharing)

不仅仅是索要文件，分享文件也有讲究。

### A. 文件请求 (索要文件)
优雅地向他人索要文件。

*   **场景**：收集 50 个人的作业/照片。
*   **操作**：
    1.  右键文件夹 > **创建文件请求**。
    2.  **密码保护**：必须设置，防止链接泄露。
    3.  **有效期**：设置截止时间。
    4.  **你的姓名**：让上传者知道发给谁。
*   **体验**：对方打开链接 -> 输入名字 -> 拖入文件 -> 上传。**对方看不到文件夹里已有的内容**，保护隐私。

### B. 分享文件 (共享链接)
把 NAS 里的文件发给朋友，而不是传到微信再发。

*   **操作**：右键文件/文件夹 > **共享**。
*   **安全设置 (关键)**：
    *   **有效期**：设置链接 1 天后失效，防止永久泄露。
    *   **密码**：防止链接被爬虫抓取。
    *   **访问次数**：设置为 "1"，下载一次即失效（阅后即焚）。
*   **Gofile.me**：如果你开启了 QuickConnect，生成的链接是 `gofile.me` 开头的，公网可直接访问。

## 6. 权限管理 (ACL) 进阶

*   **禁止下载**：
    *   右键文件夹 > 属性 > 权限 > 高级选项 > **禁止下载文件**。
    *   **效果**：用户只能在线预览（如看图、看 PDF），不能右键下载。
    *   **注意**：这只对 File Station 有效。如果用户通过 SMB 访问，还是能拷走。
*   **查看者 (Viewer)**：
    *   File Station 内置了多种查看器。
    *   **图片**：支持 RAW 格式预览。
    *   **视频**：支持在线播放（取决于浏览器解码能力）。
    *   **文档**：支持 PDF, Office 文档预览（需安装 Document Viewer）。

## 7. 虚拟盘与 ISO

*   **装载 ISO**：右键 `.iso` 文件 > 装载虚拟驱动器。直接读取光盘内容，无需解压。
*   **装载文件夹**：(仅限命令行) `mount --bind /volume1/a /volume1/b`，可以把一个文件夹映射到另一个位置（如把下载目录映射到 Web 目录）。

## 8. 进阶技巧 (Pro Tips)

### A. 解压缩乱码终极解决方案
Windows 下压缩的 zip 包，在 NAS（Linux）上解压经常出现文件名乱码。
1.  **File Station 设置**：
    *   **设置** > **常规** > **代码页**。
    *   **非 Unicode 语言**：选择 **简体中文 (GB18030)**。
    *   **应用**：再次尝试解压，乱码消失。
2.  **密码支持**：
    *   File Station 支持解压带密码的 zip/7z/rar 文件。
    *   **自动解压**：可以设置在特定文件夹（如下载目录）自动解压（需配合脚本或第三方工具）。

### B. 回收站策略 (#recycle)
防止“手滑”误删文件。
1.  **启用**：控制面板 > 共享文件夹 > 编辑 > 启用回收站。
2.  **访问**：只有管理员组（administrators）默认能看到 `#recycle` 文件夹。普通用户需要单独授权。
3.  **自动清空**：
    *   **任务计划**：控制面板 > 任务计划 > 新增 > 回收站。
    *   **策略**：设置每周末自动清空 **30 天前** 删除的文件。既保留了后悔药，又不会撑爆硬盘。

### C. 收藏夹 (Favorites)
如果你有几十个文件夹，每次都要点好几层才能找到常用的。
*   **操作**：进入深层目录 > 点击菜单栏的 **☆ (收藏)** > 添加到收藏夹。
*   **效果**：该文件夹会直接出现在 File Station 左侧栏的最上方，一键直达。

### D. 移动端联动 (DS file)
*   **照片备份**：虽然 Photos 更专业，但 DS file 的“照片备份”功能更简单粗暴，直接把手机相册同步到指定文件夹。
*   **离线文件**：在有 Wi-Fi 时把视频/文档下载到手机本地，飞机上也能看。


## 8. 智能拖拽与日志

*   **覆盖策略**：当你把重名文件拖进去时，DSM 7 会弹窗询问：
    *   **覆盖**：替换旧文件。
    *   **跳过**：不复制。
    *   **重命名**：自动重命名为 `file(1).txt`。
*   **队列策略**：如果你同时拖入 100 个大文件，File Station 会自动排队，默认同时进行 1-2 个任务，避免磁盘 IO 爆炸。你可以在后台任务管理器中调整并发数。
*   **启用日志**：
    *   **设置 > 常规 > 启用 File Station 日志**。
    *   **作用**：记录谁在什么时候删除了什么文件。这对于团队协作或家庭共享非常重要，方便追溯误删。

## 9. Docker 配置文件修改技巧

File Station 是修改 Docker 配置文件的神器。

*   **直接编辑**：
    *   右键 `.yaml`, `.json`, `.conf` 文件 > 用文本编辑器打开。
    *   **注意**：DSM 的文本编辑器支持语法高亮，但不支持 YAML 格式校验。修改前**务必备份**。
*   **权限问题**：
    *   如果 Docker 容器生成的配置文件（如 MySQL 的 `my.cnf`）是 `root` 权限，你用普通管理员账号可能无法保存。
    *   **解决**：
        1.  使用 SSH 修改权限：`sudo chown admin:users filename`。
        2.  或者在 File Station 中右键 > 属性 > 权限，尝试夺取所有权（通常需要 admin 组权限）。
