# DSM 内存检查与优化

内存不足会导致系统卡顿、Docker 容器崩溃或文件传输变慢。以下是检查和优化内存的方法。

## 1. 快速检查：桌面微件

- **DSM 7.x**：点击桌面右上角的“系统状况”图标（小部件），将鼠标悬停在“内存”部分，可以看到实时的内存使用率。

## 2. 深度分析：资源监控

打开“资源监控”套件，可以获得更详细的信息。

### 内存构成分析
在“资源监控” > “内存”标签页中，你会看到内存被分为几类：
- **已用 (Used)**：被系统和运行中的程序实际占用的内存。
- **缓存 (Cached)**：系统为了加速文件读取而预加载的数据。**注意：这部分内存虽然显示被占用，但在程序需要时会自动释放，所以通常不需要担心。**
- **可用 (Free)**：完全空闲的内存。
- **保留 (Reserved)**：系统保留的内存。

### 查找内存大户
1.  进入“任务管理器” > “进程”。
2.  点击“内存”列标题进行排序。
3.  常见的内存大户包括：
    - `synology-drive` (Synology Drive Server)
    - `docker` 相关的进程
    - `postgres` (数据库服务)

## 3. 内存优化建议

1.  **停用不常用的套件**：进入“套件中心”，停用那些你几个月都不用一次的套件。
2.  **限制 Docker 容器内存**：在 Container Manager (Docker) 中，编辑容器设置，在“资源限制”中设置内存上限，防止单个容器吃光所有内存。
3.  **增加物理内存**：如果你的 NAS 支持内存扩展（如 Plus 系列），且长期内存占用率（不含缓存）超过 70-80%，建议购买兼容的内存条进行升级。
4.  **重启系统**：长时间运行后，重启 NAS 可以释放一些僵尸进程占用的资源。
