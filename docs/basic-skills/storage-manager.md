# 存储管理器深度解析

存储管理器 (Storage Manager) 是 DSM 的心脏。理解 RAID、Btrfs 和数据清洗，能让你在硬盘故障时不再慌张。

## 1. RAID 类型选择指南

| RAID 类型 | 硬盘数 | 容错 | 空间利用率 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **SHR (Synology Hybrid RAID)** | 1+ | 1-2 块 | **最优** (支持混插) | **99% 家用/中小企业首选**。支持不同容量硬盘混用，升级扩容最灵活。 |
| **Basic** | 1 | 0 | 100% | 不推荐。单盘运行，坏了数据全丢。仅用于做临时缓存盘。 |
| **RAID 1** | 2 | 1 块 | 50% | 只有两盘位且追求数据绝对安全时。 |
| **RAID 5** | 3+ | 1 块 | (N-1)/N | 传统企业方案。**缺点**：所有硬盘容量必须一致，扩容受限。 |
| **RAID 6** | 4+ | 2 块 | (N-2)/N | 追求极高安全性，允许同时坏 2 块盘。 |
| **RAID 10** | 4+ | 1+ 块 | 50% | 追求极致读写速度（数据库/虚拟化），成本高。 |

**结论**：闭眼选 **SHR**。它是群晖的杀手锏，允许你先买两块 4T，以后随时加两块 8T，空间一点不浪费。

## 2. Btrfs vs EXT4

DSM 7 默认推荐 Btrfs，理由如下：

*   **文件自愈 (Self-healing)**：Btrfs 存储了数据和元数据的校验和 (Checksum)。读取文件时如果发现数据损坏（位衰减），系统会自动利用 RAID 镜像修复数据，**这是 EXT4 做不到的**。
*   **快照 (Snapshot)**：秒级备份，防勒索病毒神器。
*   **存储空间回收**：支持在线回收未使用的空间。

**结论**：除非你是极低端的 ARM 机型不支持 Btrfs，否则**必选 Btrfs**。

## 3. 数据清理 (Data Scrubbing)

硬盘没坏，数据就一定安全吗？**位衰减 (Bit Rot)** 是隐形杀手。磁性介质长时间不读写，磁性可能减弱导致个别 0 变成 1。

*   **作用**：全盘扫描并比对校验和，发现并修复潜在的数据不一致。
*   **建议**：
    *   **频率**：每 3-6 个月运行一次。
    *   **设置**：存储管理器 > 存储池 > 计划数据清理。
    *   **注意**：清理过程会占用大量 IO，建议设置在半夜进行。

## 4. 硬盘休眠机制

*   **利弊**：休眠能省电、降温，但频繁的启停（每天几十次）最伤硬盘电机。
*   **建议**：
    *   如果你的 NAS 跑了 Docker、P2P 下载、Web 服务，**硬盘是无法休眠的**，建议直接关闭休眠功能。
    *   如果是纯冷备份机（一周开一次），可以开启休眠。

## 5. 坏道与 SMART 检测

DSM 会自动检测硬盘健康，但有时会误报或漏报。

*   **快速检测**：读取硬盘自带的 SMART 日志，几秒钟完成。建议每月一次。
*   **完整检测**：全盘扫描坏块，耗时数小时。建议每半年一次，或怀疑硬盘有问题时。
*   **关键指标**：
    *   **ID 1 (Raw Read Error Rate)**：读取错误率（希捷正常值通常很高，西数应为 0）。
    *   **ID 5 (Reallocated Sector Count)**：重映射扇区计数。**只要不为 0，立马换盘**。
    *   **ID 197 (Current Pending Sector)**：当前待映射扇区。预警信号。

## 6. SSD 缓存避坑

*   **读写缓存 (RAID 1)**：提升读写性能，但如果 SSD 意外掉线或损坏，**可能会导致存储池损毁，数据丢失**。建议使用企业级 SSD，并配置 UPS。
*   **只读缓存 (RAID 0)**：只加速读取。**非常安全**，SSD 坏了数据不会丢，只是缓存失效而已。
*   **建议**：家用环境（看电影、存照片）**只读缓存**足够了，甚至不需要缓存（瓶颈在千兆网卡）。只有跑虚拟机、数据库、大量小文件读写时，缓存才有明显效果。
