# 文件服务 (File Services) 深度调优：速度与兼容性

文件服务是 NAS 的本职工作。很多用户买了 2.5G 网口甚至万兆网卡的 NAS，却发现传输速度只有几十 MB/s，或者 Mac 总是连不上，问题通常出在文件服务的配置上。

## 1. SMB 服务：Windows 与 Mac 的首选

SMB (Server Message Block) 是目前最通用的文件共享协议。无论是 Windows、Mac 还是 Linux，首选 SMB。

### A. 协议版本设置 (关键)
*   **路径**：控制面板 > 文件服务 > SMB > 高级设置。
*   **最大 SMB 协议**：**SMB3**。
*   **最小 SMB 协议**：**SMB2**。
    *   **警告**：千万不要开启 **SMB1**，除非你有十几年前的打印机或 XP 系统。SMB1 极不安全，是勒索病毒的温床。

### B. 开启 SMB 多通道 (Multichannel) —— 速度翻倍！
如果你有两个千兆网口，或者电脑和 NAS 都有双网口，开启此功能可以让速度叠加（例如 1000Mbps + 1000Mbps = 200MB/s）。
*   **条件**：NAS 和电脑都需要有多张网卡连接到同一个交换机/路由器。
*   **设置**：在 SMB 高级设置 > 其他 > 勾选 **启用 SMB3 多通道**。
*   **注意**：这比“链路聚合 (Link Aggregation/LACP)”更灵活，不需要交换机支持，配置更简单。

### C. 兼容性设置
*   **Mac 用户**：
    *   **启用 Bonjour Time Machine 广播**：让 Mac 的时间机器能找到 NAS。
    *   **禁止 .DS_Store 文件写入**：虽然不是 NAS 的设置（需在 Mac 终端执行），但能避免 NAS 文件夹里全是垃圾文件。
*   **Windows 用户**：
    *   **启用 WS-Discovery**：让 NAS 出现在 Windows 资源管理器的“网络”邻居里，而不是只能通过 IP 访问。

### D. 传输日志 (审计)
想知道谁删除了公司的重要文件？
*   **设置**：SMB 设置 > 勾选 **启用传输日志**。
*   **选项**：勾选“删除”和“写入”。
*   **查看**：在 **日志中心** > **文件传输日志** 中查看。
*   **注意**：开启日志会轻微影响小文件读写性能（约 5-10%），家用可不比开，公司必开。

## 2. AFP 服务：该说再见了

AFP (Apple Filing Protocol) 是苹果早期的协议。

*   **现状**：苹果官方都已经弃用 AFP，转向 SMB3。
*   **建议**：**彻底关闭 AFP 服务**。
*   **理由**：
    1.  SMB3 在现代 macOS 上速度更快、更稳定。
    2.  同时开启 AFP 和 SMB 会导致文件锁定冲突（比如你用 SMB 打开了文档，别人用 AFP 却能强制覆盖）。
    3.  Time Machine 备份现在也完美支持 SMB。

## 3. NFS 服务：Linux 与 媒体播放器的最爱

如果你用 Kodi、Infuse、Plex 或者 Linux 服务器挂载 NAS，NFS (Network File System) 是效率最高的选择。

### A. 为什么用 NFS？
*   **低开销**：没有复杂的身份验证，CPU 占用极低。
*   **高吞吐**：非常适合局域网内传输 4K 原盘电影。

### B. 权限配置 (难点)
NFS 不像 SMB 那样输入账号密码，它通过 IP 地址和 UID/GID 验证。
*   **设置**：控制面板 > 共享文件夹 > 编辑某文件夹 > **NFS 权限** > 新增。
*   **客户端 IP**：填入播放器的 IP（如 `192.168.1.100`）或网段（`192.168.1.0/24`）。
*   **Squash (映射)**：
    *   **无映射 (No mapping)**：客户端是 root 就拥有 NAS 的 root 权限（极不安全，但最方便）。
    *   **映射所有用户为 admin (Map all users to admin)**：**推荐**。无论客户端是谁，在 NAS 上都视为 admin 操作，避免权限不足导致无法读取/写入字幕。
    *   **映射所有用户为 guest**：最安全，只有只读权限。
*   **安全性**：NFS 默认基于 IP 信任，**千万不要把 NFS 端口 (2049) 暴露到公网**。

## 4. FTP/FTPS/SFTP：老当益壮

虽然 Web 界面很方便，但 FTP 在某些场景下仍不可替代（如傻瓜式监控摄像头、老式扫描仪）。

*   **FTP (端口 21)**：明文传输，极不安全。**仅限局域网内老设备使用**。
*   **FTPS (SSL/TLS)**：加密的 FTP。推荐开启。
*   **SFTP (SSH File Transfer Protocol)**：基于 SSH 协议，端口默认 22。
    *   **优点**：只需要开一个 SSH 端口就能传输文件，防火墙配置简单。
    *   **缺点**：速度受限于 CPU 加密解密性能，通常比 FTP 慢。

## 5. WebDAV：远程挂载神器

想在公司电脑像操作本地磁盘一样操作家里的 NAS？WebDAV 是最佳方案。

*   **套件**：WebDAV Server。
*   **端口**：HTTPS 默认 5006。
*   **客户端推荐**：
    *   **Windows**: RaiDrive (免费版即可，挂载为 Z 盘)。
    *   **Mac**: Finder (原生支持，但性能一般) 或 Mountain Duck。
    *   **iOS/Android**: FE File Explorer, Documents。
*   **技巧**：配合 DDNS 和 SSL 证书，WebDAV 是最安全的公网文件管理方式之一。

## 6. rsync：专业备份协议

如果你有两台 NAS 需要同步，或者从 Linux 服务器备份数据到 NAS。
*   **路径**：控制面板 > 文件服务 > rsync。
*   **用途**：主要用于 **Hyper Backup** 的 rsync 目标，或 **Synology Drive ShareSync**。
*   **安全性**：建议修改默认端口（873），并配置允许访问的 IP 列表。
