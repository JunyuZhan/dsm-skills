# USB Copy 深度指南：摄影师与数据搬运工的神器

虽然现在的 NAS 都在讲究网络同步，但有时候 **USB 物理连接** 才是最快、最稳的。USB Copy 是群晖最容易被忽视的官方套件，它能让你的移动硬盘变成自动备份中心。

## 1. 为什么需要 USB Copy？

*   **摄影师**：每次拍完几百张照片，直接插上 SD 卡读卡器，听到“滴”的一声，照片自动备份到 NAS，拔卡走人。不需要开电脑，不需要手动复制。
*   **冷备份**：定期把重要数据（如宝宝照片）导出到移动硬盘，断网保存，防勒索病毒。
*   **数据导入**：新买了 NAS，旧移动硬盘里有 4TB 数据，直接插上 NAS 导入，比千兆网络快几倍。

## 2. 三种模式详解

打开 USB Copy，创建任务时有三种模式：

*   **照片/视频导入 (Photo/Video Import)**：
    *   **逻辑**：只复制 DCIM 文件夹里的媒体文件。
    *   **智能**：按拍摄日期自动重命名文件夹（如 `2023-10-01`），自动整理乱七八糟的 SD 卡结构。
    *   **适用**：相机 SD 卡备份。
*   **数据导入 (Data Import)**：
    *   **逻辑**：把 USB 设备里的**所有**文件复制到 NAS 指定文件夹。
    *   **增量**：第一次全量，以后只复制新增/修改的文件。
    *   **适用**：把移动硬盘内容归档到 NAS。
*   **数据导出 (Data Export)**：
    *   **逻辑**：把 NAS 指定文件夹复制到 USB 设备。
    *   **镜像**：如果你勾选了“删除目标中不存在的文件”，它就是一个完美的单向同步备份。
    *   **适用**：冷备份重要数据到移动硬盘。

## 3. 自动化触发 (Trigger)

这是 USB Copy 的灵魂。

*   **插上即运行**：
    *   勾选 **插入 USB/SD 设备时立即复制**。
    *   **效果**：只要插上这个特定的 U 盘（系统会记录设备 ID），任务自动开始。
*   **复制完成后弹出**：
    *   勾选 **复制完成后卸载 USB/SD 设备**。
    *   **效果**：复制完后 NAS 会“滴”一声（需开启蜂鸣器通知），硬盘灯灭，你直接拔掉即可。
    *   **注意**：一定要等它卸载（灯灭或听到提示音），否则数据会损坏。

## 4. 文件冲突策略

*   **重命名**：如果 NAS 里已经有同名文件（但内容不同），自动重命名为 `file_1.jpg`。
*   **覆盖**：总是用 USB 里的新文件覆盖 NAS 里的旧文件。
*   **跳过**：如果同名，不复制。

## 5. 多版本备份 (Versioning)

虽然 USB Copy 不像 Hyper Backup 那么强大，但它也支持简单的版本控制。

*   **启用**：勾选 **当目标存在同名文件时，保留两个版本**。
*   **效果**：旧文件会被重命名为 `file_old.jpg`，新文件保留原名。
*   **注意**：它不支持复杂的历史回溯，如果需要多版本，请使用 Hyper Backup 的 **Local Folder & USB** 模式（虽然不能自动插拔触发，但版本管理更强）。

## 6. 性能优化

*   **文件系统**：
    *   **推荐**：移动硬盘格式化为 **exFAT** 或 **NTFS**。
    *   **注意**：exFAT 需要在套件中心安装 **exFAT Access**（DSM 7 免费，DSM 6 收费）。
    *   **EXT4**：性能最好，但 Windows 电脑读不出来。
*   **USB 接口**：
    *   **前置 USB**：通常是 USB 2.0 或 3.0，适合插 SD 卡或 U 盘。
    *   **后置 USB**：通常也是 USB 3.0，更稳定，适合插大容量移动硬盘。
    *   **注意**：如果移动硬盘是 3.5 寸的，**必须接独立电源**，NAS 的 USB 供电带不动。

## 7. 进阶玩法：作为“钥匙”

你可以设置一个任务：
1.  插入特定 U 盘。
2.  自动把 NAS 里的 `KeyPass` 数据库或 `ssh` 密钥导出到 U 盘。
3.  拔掉 U 盘。

这样你的敏感数据只有在你插入物理密钥时才会更新到随身携带的设备上，非常安全。
