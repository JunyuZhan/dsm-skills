# 数据加密与密钥管理

数据安全不仅是防黑客，还要防物理失窃。如果 NAS 被小偷偷走，硬盘被拔出来挂载到 Linux 电脑上，你的数据会直接暴露吗？

## 1. 共享文件夹加密

DSM 支持基于 **eCryptfs** 的文件夹级加密。

### 创建加密文件夹
1.  控制面板 > 共享文件夹 > 新增。
2.  在“加密”选项卡中，勾选 **加密此共享文件夹**。
3.  输入加密密钥（密码）。
4.  系统会提示你下载一个 `.key` 文件，**务必保存好！**

### 访问方式
*   **已挂载**：输入密码挂载后，文件夹像普通文件夹一样读写，性能略有损耗（AES-NI 指令集 CPU 损耗很小）。
*   **未挂载**：文件夹是锁定的，看不到任何内容。
*   **重启后**：默认处于**未挂载**状态。

## 2. 密钥管理器 (Key Manager)

每次重启都要手动输入密码挂载很麻烦，但不挂载又导致自动化服务（如 Docker）失败。**密钥管理器**解决了这个矛盾。

### 配置
1.  控制面板 > 共享文件夹 > 密钥管理器。
2.  **系统分区**：将密钥存储在系统分区。这样重启后可以自动挂载。
    *   *风险*：如果整机被偷，小偷开机就能看到数据。
3.  **USB 闪存盘 (推荐)**：
    *   插入一个闲置的 USB U 盘。
    *   在密钥管理器中，选择将密钥存储在 USB 盘。
    *   设置“开机自动挂载”。

### 物理密钥玩法
*   **日常**：U 盘一直插在 NAS 后背。
*   **离家**：出远门时，**拔走 U 盘**。
*   **效果**：此时 NAS 如果被偷，小偷开机后，因为没有 U 盘里的密钥，加密文件夹无法挂载，数据是一堆乱码。**这才是真正的物理安全。**

## 3. 加密对性能的影响

*   **文件名加密**：文件名最长被限制在 143 个字符（原为 255）。
*   **吞吐量**：在 J4125 等现代 CPU 上，AES-NI 硬件加速非常给力，千兆网络下几乎感觉不到性能差异。但在万兆网络下，可能会有 20-30% 的损耗。
*   **限制**：加密文件夹**不支持 NFS 协议**（部分旧版本限制，新版本已支持但配置复杂），建议使用 SMB。

## 4. 备份加密数据

*   **Hyper Backup**：备份加密文件夹时，可以选择“保留加密状态”备份。这样备份在云端的数据也是加密的，即使云服务商泄露数据，你的隐私也是安全的。

## 5. 整卷加密 (Volume Encryption)

DSM 7.2 引入了重磅功能：**整个存储空间加密**。

*   **区别**：以前是基于文件夹 (eCryptfs)，现在是基于卷 (LUKS)。
*   **优势**：
    *   性能更好。
    *   支持文件名超长。
    *   支持 NFS 协议。
    *   管理更统一。
*   **建议**：如果你是 DSM 7.2 以上的新机器，**优先使用整卷加密**。但老机器升级上来无法直接转换，需要新建存储池并迁移数据。
