# Hyper Backup 深度指南：不仅是备份

Hyper Backup 是群晖最核心的数据保护工具。大多数人只用它来做简单的“每日备份”，但它其实拥有企业级的特性，如**块级增量备份**、**跨版本重删**、**完整性检查**和**客户端加密**。

## 1. 备份机制：块级增量与重删

Hyper Backup 不是简单的文件复制。
*   **块级增量 (Block-level Incremental)**：如果你修改了一个 10GB 视频文件的元数据（如标题），它不会重新上传 10GB，只会上传修改的那几 KB 数据块。
*   **跨版本重删 (Deduplication)**：如果你今天备份了文件 A，明天把 A 复制了一份变成 B，Hyper Backup 能够识别出 A 和 B 是相同的数据块，物理存储只占一份空间。

**优势**：极大地节省备份空间和带宽。你可以放心地保留 256 个历史版本，而不用担心空间爆炸。

## 2. 智能循环策略 (Smart Recycle)

“我想保留很久的备份，但不想存太多版本。”
不要使用“从最早版本开始删除”，推荐使用 **Smart Recycle**。

系统会自动根据规则保留版本：
*   **过去 24 小时**：每小时保留一个版本。
*   **过去 1 个月**：每天保留一个版本。
*   **超过 1 个月**：每周保留一个版本。

这样你既能找回刚才误删的文件，也能找回半年前的文件，而且版本数量可控。

## 3. 完整性检查 (Integrity Check)

备份文件坏了比没有备份更可怕。Hyper Backup 提供了**数据完整性检查**功能。

*   **检查索引**：验证数据库索引是否一致（速度快）。
*   **检查数据**：读取实际的数据块并校验 Checksum（速度慢，但可靠）。
*   **建议**：设置每周检查索引，每月检查数据（可以设置时间限制，如“每次检查 30 分钟”，系统会轮流检查所有数据）。

## 4. 客户端加密 (Client-side Encryption)

当你把数据备份到公有云（百度网盘、OneDrive、Synology C2）时，隐私是最大的顾虑。

*   **启用**：在创建任务时勾选“启用客户端加密”。
*   **密钥**：设置密码，并下载 `*.pem` 密钥文件。
*   **安全性**：数据在离开 NAS 之前就已经被 AES-256 加密。**即使是云服务商也无法看到你的照片和文档**。
*   **注意**：**千万不要弄丢密钥**！没有密钥，神仙也救不回数据。

## 5. Hyper Backup Explorer：无需还原即可浏览

你想查看备份里的某张照片，但不想把整个 500GB 的备份都还原回来？

*   **Windows/Mac/Linux**：下载 **Hyper Backup Explorer** 桌面工具。
*   **使用**：
    *   **本地浏览**：插上 USB 备份盘，用工具打开 `.hbk` 文件，可以直接浏览目录，右键提取单个文件。
    *   **云端浏览**：登录 Synology C2，直接在线浏览和下载。

## 6. HB Vault：NAS 对 NAS 备份

如果你有两台群晖（一台在公司，一台在家），可以互为备份。

1.  **接收端 (目标 NAS)**：
    *   安装 **Hyper Backup Vault** 套件。
    *   确保防火墙放行端口 6281。
2.  **发送端 (源 NAS)**：
    *   Hyper Backup > 新增 > **远程 NAS 设备**。
    *   输入目标 NAS 的 IP/域名。
    *   此时目标 NAS 的 Vault 中会显示接收到的数据统计。

## 7. 单个文件还原 (File-level Restore)

在 DSM 中，不需要恢复整个任务。
1.  打开 **Hyper Backup**。
2.  点击左侧的 **备份资源管理器** (放大镜图标)。
3.  下方有时间轴，拖动选择时间点。
4.  找到文件，右键 **复制到...** 或 **下载**。

## 8. 故障排除

*   **备份挂起/极慢**：
    *   检查是否开启了大量小文件的备份（如 `node_modules`）。建议打包成压缩包再备份。
    *   检查上行带宽是否跑满。
*   **空间回收慢**：
    *   删除大量版本后，空间不会立即释放。系统需要在后台进行数据整理（Compact），这可能需要几小时甚至几天，期间磁盘 I/O 会很高。
