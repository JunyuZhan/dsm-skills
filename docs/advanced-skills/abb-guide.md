# Active Backup for Business (ABB) 深度指南

Active Backup for Business (ABB) 是群晖最杀手级的套件之一。它能免费备份 Windows PC、物理服务器、文件服务器和虚拟机，支持全局重删，极大地节省存储空间。

## 为什么选择 ABB？

*   **免费商用**：无需购买许可证，支持无限台设备（受限于 NAS 性能）。
*   **全局重删**：跨设备去重。例如 10 台 Win10 电脑备份，系统文件只存一份，节省 50%+ 空间。
*   **整机备份**：不仅备份文件，还备份系统。电脑坏了？直接还原整个系统，无需重装软件。
*   **增量备份**：CBT 技术，只传输变化的数据块，速度极快。

## 实战 1：备份 Windows PC

这是最常用的功能，保护公司或家庭的电脑。

1.  **NAS 端**：
    *   打开 ABB 套件，激活（需登录 Synology 账户）。
    *   进入 **PC/Mac** > **模板**，创建一个默认模板（设置备份计划、保留策略）。
2.  **电脑端**：
    *   下载并安装 **Synology Active Backup for Business Agent**。
    *   输入 NAS IP、用户名、密码。
    *   连接成功后，会自动套用模板开始备份。
3.  **恢复**：
    *   **文件级恢复**：在 NAS 上打开 **Active Backup for Business Portal**，像浏览网盘一样找回误删文件，支持网页端直接下载或还原。
    *   **整机恢复**：需要制作 **Synology Restore Media**（恢复 U 盘）。电脑插上 U 盘启动，连接 NAS，选择时间点还原。

---

## 实战 2：备份物理服务器/虚拟机

对于运行关键业务的 Windows Server 或 Linux 服务器。

*   **物理服务器**：步骤同 PC，但在 ABB 中归类为“物理服务器”，支持应用感知的备份（如 VSS，确保 SQL Server 数据一致性）。
*   **虚拟机 (VMware/Hyper-V)**：
    *   无需在虚拟机内安装 Agent。
    *   直接在 ABB 中添加 vCenter 或 Hyper-V 宿主机信息。
    *   ABB 会调用虚拟化平台的快照接口进行备份。
    *   **黑科技**：支持 **即时还原到 Synology VMM**。如果生产环境服务器挂了，可以直接在 NAS 的 VMM 虚拟机中把备份镜像跑起来，几分钟内恢复业务。

---

## 实战 3：备份文件服务器 (SMB/Rsync)

如果你有另一台 NAS（非群晖）或 Linux 文件服务器需要备份。

1.  **文件服务器** > **添加服务器**。
2.  选择 SMB 或 Rsync。
3.  输入地址和账号。
4.  **模式选择**：
    *   **多版本**：每次备份都保留一个快照，占用空间较多。
    *   **镜像**：始终与源端保持一致（源端删，备份端也删），仅保留最新版。
    *   **增量**：推荐。保留历史版本，且利用重删技术。

## 3-2-1 备份策略集成

ABB 只是把数据备份到了 NAS 上，如果 NAS 坏了怎么办？需要结合 **Hyper Backup** 实现 3-2-1。

1.  **第一步 (3)**：ABB 将 PC/服务器数据备份到 NAS（生产数据 + 1个备份）。
2.  **第二步 (2)**：使用 Hyper Backup，将 ABB 的备份数据（位于 `/ActiveBackupforBusiness` 共享文件夹）备份到 USB 硬盘。
    *   *注意：备份 ABB 数据时，Hyper Backup 会自动检测并备份重删数据库。*
3.  **第三步 (1)**：使用 Hyper Backup 将数据备份到云端（Synology C2）或异地 NAS。

## 常见问题与优化

*   **备份速度慢**：
    *   检查网络是否千兆/万兆。
    *   首次备份是全量，后续是增量。
    *   关闭杀毒软件对 Agent 的扫描。
*   **占用空间过大**：
    *   检查保留策略。建议开启“版本保留”，例如保留最近 7 天，每周 4 个，每月 12 个。
    *   确认为 Btrfs 文件系统（必须）。
*   **电脑休眠导致备份失败**：
    *   ABB Agent 可以唤醒电脑备份（需 BIOS 开启 Wake-on-LAN），或者设置备份时阻止休眠。
