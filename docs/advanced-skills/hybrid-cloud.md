# 混合云架构实战 (Hybrid Cloud)

不要把鸡蛋放在同一个篮子里。将本地 NAS 与公有云结合，打造“永不掉线”的数据中心。

## 1. Cloud Sync：云端硬盘本地化

**功能**：将百度网盘、阿里云盘、OneDrive、Google Drive 挂载到 NAS，并保持双向同步。

### 实战技巧
*   **单向上传 (备份)**：将重要照片备份到 Google Drive。设置“仅上传本地更改”，这样即使云端被误删，本地文件还在。
*   **百度网盘下载**：虽然 Cloud Sync 同步百度网盘速度一般，但用来**通宵下载**大文件非常方便。在电脑上把文件丢进网盘的“我的应用数据/Cloud Sync”目录，NAS 就会自动开始下载。
*   **加密上传**：不信任云服务商？在创建任务时勾选“数据加密”。文件上传前会被 NAS 加密，云端看到的是乱码，下载回 NAS 时自动解密。

## 2. Hyper Backup + S3/Glacier：低成本冷备份

**场景**：你需要一个异地灾备，但不想再买一台 NAS。

### 方案 A：Synology C2 (官方云)
*   **优点**：集成度最高，速度极快，支持版本去重。
*   **缺点**：价格略贵（约 $60/年/1TB）。

### 方案 B：Amazon S3 Glacier Deep Archive (极致低成本)
*   **适合**：存放几年都不看一眼，但又不能丢的归档数据（如 10 年前的项目文档）。
*   **成本**：存储费用极低（约 $1/月/1TB），但**取回费用很高**。
*   **配置**：在 Hyper Backup 中选择 S3 存储，存储类选择 `Glacier Deep Archive`。

### 方案 C：Backblaze B2
*   **性价比之王**。存储费用低，且下载流量有免费额度。支持 S3 协议，完美兼容 Hyper Backup。

## 3. Hybrid Share：云端缓存加速

**场景**：公司在上海和北京都有分部，需要共享大量设计素材。

**原理**：
1.  数据存放在 Synology C2 云端。
2.  上海和北京的 NAS 分别挂载这个 C2 空间。
3.  **本地缓存**：经常访问的文件会被自动缓存在本地 NAS 的 SSD/HDD 中，访问速度是局域网速度。不常用的文件只在云端保留元数据。
4.  **效果**：实现了“云端无限空间” + “本地极速体验”。

## 4. Active Backup for Google Workspace / Microsoft 365

**场景**：担心 Google 封号或者员工离职删库？

**功能**：
*   自动备份公司所有员工的 Gmail、Drive、日历、联系人。
*   **免授权费**：只要你的 NAS 硬盘够大，备份多少账号都免费。
*   **还原**：支持细颗粒度还原，甚至可以把离职员工的邮件还原到新员工的邮箱里。

## 5. Docker + Rclone：万能云挂载

如果 Cloud Sync 不支持某些小众网盘（如 Mega, 115），Rclone 是终极方案。

1.  **部署**：使用 Docker 部署 `rclone/rclone`。
2.  **配置**：通过 Web UI 配置网盘。
3.  **挂载**：使用 `rclone mount` 命令将网盘挂载为 NAS 的本地文件夹。
4.  **联动**：挂载后，Plex/Jellyfin 可以直接读取网盘里的电影进行播放。
