# 数据迁移与灾难恢复指南

数据是 NAS 的核心。无论是更换新设备，还是应对硬盘故障，掌握正确的数据迁移和恢复方法至关重要。

## 三种主流迁移方式对比

群晖官方提供了三种主要的迁移方式，适用于不同的场景。

### 1. 硬盘迁移 (HDD Migration)

**最快，但有限制。**

*   **原理**：直接将旧 NAS 的硬盘拔出，按顺序插入新 NAS。
*   **适用场景**：
    *   旧 NAS 坏了，但硬盘没坏。
    *   升级到盘位更多或同系列的机型。
*   **限制**：
    *   **不能降级**：新机型必须比旧机型更新或同级（例如不能从 DS920+ 迁移到 DS218+）。
    *   **盘位限制**：新 NAS 盘位必须 >= 旧 NAS。
    *   **系统限制**：某些高端系列（XS/XS+）不支持从低端系列（J/Play）直接插盘迁移。
*   **步骤**：
    1.  备份配置（控制面板 > 更新和还原 > 配置备份）。
    2.  关机，拔盘。
    3.  **按原顺序**插入新 NAS。
    4.  开机，通过 find.synology.com 查找，状态会显示“可迁移”。
    5.  按提示安装 DSM（选择“保留数据和设置”）。

### 2. Migration Assistant

**最平滑，服务不中断。**

*   **原理**：通过局域网，将一台 NAS 的所有数据、设置、套件“克隆”到另一台 NAS。
*   **适用场景**：
    *   新旧两台 NAS 都能正常开机。
    *   想要最完整的系统迁移（包括 Docker 容器、套件设置）。
*   **优点**：迁移过程中，旧 NAS 的服务（如 SMB、Web）仍可访问。
*   **缺点**：非常耗时（取决于数据量和网络速度）。
*   **步骤**：
    1.  在新 NAS 上安装 **Migration Assistant** 套件。
    2.  打开套件，按向导连接旧 NAS。
    3.  系统会自动同步数据。

### 3. Hyper Backup 备份还原

**最灵活，跨文件系统。**

*   **原理**：先将数据备份到外部设备（USB硬盘）或另一台 NAS，再在新 NAS 上还原。
*   **适用场景**：
    *   想从 EXT4 文件系统转换为 Btrfs 文件系统（插盘迁移无法更改文件系统）。
    *   跨度很大的机型升级。
*   **步骤**：
    1.  旧 NAS：使用 Hyper Backup 备份数据、配置、套件到 USB 硬盘。
    2.  新 NAS：安装 Hyper Backup，选择“还原”，从 USB 硬盘恢复。

## 灾难恢复策略

不要等到数据丢失才后悔。建立完善的恢复策略是每个 NAS 用户的必修课。

### 3-2-1 备份法则

*   **3** 份数据副本（原件 + 2个备份）。
*   **2** 种不同的存储介质（NAS + USB硬盘 / NAS + PC）。
*   **1** 个异地备份（云端 / 异地 NAS）。

### 核心恢复工具

#### 1. Snapshot Replication (快照)

*   **对抗勒索病毒的神器**。
*   **特点**：基于 Btrfs 文件系统，秒级备份，占用空间极小。
*   **设置**：
    *   安装 **Snapshot Replication** 套件。
    *   对共享文件夹开启计划快照（建议每小时一次）。
    *   设置保留策略（如保留最近 7 天的快照）。
*   **恢复**：
    *   文件误删/中毒：直接在 File Station 右键 > 快照 > 浏览/还原，找回历史版本。

#### 2. Hyper Backup (冷备)

*   **系统级兜底**。
*   建议每周备份一次核心数据到 USB 硬盘或 Synology C2 云端。
*   开启“客户端侧加密”，确保云端数据安全。

#### 3. Active Backup for Business (整机还原)

*   如果你用 NAS 备份了 PC 或服务器。
*   制作 **Synology Restore Media** 启动 U 盘。
*   当电脑系统崩溃时，插上 U 盘启动，直接从 NAS 拉取镜像还原整个系统，无需重装 Windows。

### 紧急救援模式

如果 NAS 无法启动，但数据很重要：

1.  **Ubuntu 挂载法**：
    *   找一台台式机，制作 Ubuntu Live USB 启动。
    *   将 NAS 硬盘接在台式机上。
    *   Ubuntu 通常能自动识别 Linux RAID 分区，挂载后即可通过文件管理器拷出数据。
    *   *官方有详细教程：[如何在 PC 上恢复数据](https://kb.synology.com/zh-cn/DSM/tutorial/How_can_I_recover_data_from_my_DiskStation_using_a_PC)*
