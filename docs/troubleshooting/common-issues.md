# 常见故障排除 (Troubleshooting)

NAS 遇到问题不要慌，这里有 10 个常见故障的自救指南。

## 1. 忘记管理员密码
- **症状**：无法登录 DSM。
- **解决方法**：
    1.  找到 NAS 背面的 **RESET** 按钮。
    2.  用回形针按住约 **4 秒**，直到听到一声“哔”。
    3.  立即松开。这会重置 `admin` 密码为空，并恢复网络设置（IP 变回 DHCP）。**数据不会丢失**。
    4.  注意：如果按住超过 10 秒（听到三声哔），系统会重装 DSM！千万别按久了。

## 2. 无法访问 NAS (网络问题)
- **症状**：Synology Assistant 搜不到，网页打不开。
- **排查**：
    - 检查网线指示灯是否闪烁。
    - 检查路由器 DHCP 列表，确认 NAS 是否获取到了 IP。
    - 尝试直接用网线连接电脑和 NAS，将电脑 IP 设为 `169.254.1.1`，尝试访问 NAS。

## 3. 硬盘无法休眠
- **症状**：硬盘灯一直闪，噪音大，费电。
- **原因**：
    - 后台有服务在读写（索引、缩略图生成、Cloud Sync 同步）。
    - 开启了 QuickConnect 或 DDNS 心跳检测。
    - 安装了 Docker 容器（这是最常见原因，容器通常会持续写日志）。
- **排查**：开启“休眠日志”查看唤醒源，或拔掉网线测试是否休眠。

## 4. SMB 传输速度慢
- **症状**：千兆局域网只有 10MB/s。
- **排查**：
    - 检查网线是否为 Cat5e 或 Cat6（千兆线）。
    - 检查交换机/路由器端口是否协商在 1000Mbps。
    - 检查电脑网卡驱动。
    - 尝试关闭 SMB 加密（在文件服务 > SMB > 高级设置中）。

## 5. Docker 镜像下载失败
- **症状**：`Registry` 查询超时，下载速度 0。
- **原因**：Docker Hub 被墙。
- **解决**：配置镜像加速器（如阿里云、网易云等国内镜像源）。或者使用代理服务器（在 Docker 设置中配置 HTTP 代理）。

## 6. 存储空间损毁 (Crashed)
- **症状**：存储池状态变为“红色”，只能读不能写。
- **原因**：硬盘坏道过多，或 RAID 阵列丢失数据。
- **紧急措施**：
    1.  **立刻停止一切写入操作！**
    2.  尽快通过 File Station 或 SMB 将重要数据拷贝出来。
    3.  如果是硬盘故障，更换硬盘并修复存储池。
    4.  如果是文件系统错误，可能需要联系群晖技术支持。

## 7. CPU 占用率 100%
- **症状**：DSM 界面卡顿，响应极慢。
- **排查**：
    - 打开 **资源监控** > **任务管理器** > **服务**。
    - **Universal Search / synoelasticd**：正在建立索引。如果是新导入大量照片，这是正常的，等它跑完。
    - **Docker 容器**：某个容器失控（如 Plex 转码、Jellyfin 扫描）。
    - **Antivirus Essential**：正在扫描病毒。建议卸载，NAS 裸奔即可（只要不开 SSH 弱密码）。

## 8. 噪音过大
- **原因**：
    - **风扇**：积灰或老化。可以在控制面板 > 硬件和电源 > 风扇模式中调整为“静音模式”。
    - **硬盘**：企业级硬盘（如 HC550）本身就有“炒豆子”声，这是正常的读写寻道声。
    - **共振**：给硬盘托架贴植绒胶带，或在 NAS 底部垫减震垫。

## 9. 无法更新 DSM
- **症状**：提示“连接失败”或“存储空间不足”。
- **解决**：
    - **网络**：配置 DNS 为 `223.5.5.5` 或 `114.114.114.114`。
    - **系统分区满**：SSH 登录，检查 `/` 分区使用率。通常是某个脚本把日志写到了系统盘（参考 [高级故障排查](advanced-cases.md)）。

## 10. 外网访问不稳定
- **症状**：QuickConnect 时断时续。
- **解决**：
    - QuickConnect 经群晖服务器中转，速度慢且不稳定。
    - **强烈建议**：配置 DDNS + 端口转发（如果有公网 IP），或者使用 Tailscale / Cloudflare Tunnel（如果没有公网 IP）。
