# 音乐人与音频制作人群晖 NAS 解决方案指南

## 核心痛点与需求

对于音乐制作人、录音师和混音师来说，数据资产的管理至关重要：

*   **工程文件庞大且碎片化**：Logic Pro, Pro Tools, Cubase 的工程文件夹中包含大量音频片段（WAV/AIFF），对 I/O 性能要求高。
*   **采样库管理**：动辄几 TB 的 Kontakt 采样库、Loop 素材，需要在多台设备（录音棚 Mac、家用 PC、笔记本）间同步。
*   **版本控制**：混音过程中产生的 Mix_v1, Mix_v2, Master_Final 等多个版本需要妥善保存。
*   **Demo 交付**：需要方便快捷地将 Demo 发送给歌手或客户试听，而不是通过微信传压缩包。

## 1. 存储架构与性能调优

### 文件夹结构建议

```
/volume1/Studio
  ├── 01_Projects (活跃工程)
  │   ├── 2024
  │   │   ├── 2024-03_某某单曲制作
  │   │   │   ├── Logic_Project
  │   │   │   ├── Stems (分轨导出)
  │   │   │   ├── Reference (参考曲)
  │   │   │   └── Exports (导出成品)
  │   └── ...
  ├── 02_Sample_Libraries (采样库)
  │   ├── Kontakt
  │   ├── Loops
  │   └── SFX
  ├── 03_Software_Installers (插件安装包)
  └── 99_Archive (完结项目归档)
```

### 关键性能优化

DAW（数字音频工作站）对磁盘延迟非常敏感，甚至比视频剪辑更甚，因为音频流不能有任何爆音。

1.  **全闪存存储池 (SSD Volume)**
    *   **建议**：如果有条件，建立一个纯 SSD 的存储池用于存放 `01_Projects` 和 `02_Sample_Libraries`。
    *   **原因**：加载几十 GB 的管弦乐采样库时，机械硬盘的寻道时间会导致加载缓慢甚至播放卡顿。

2.  **万兆网络 (10GbE)**
    *   **场景**：如果你想直接在 NAS 上运行工程（不复制到本地），10GbE 是必须的。
    *   **协议**：推荐使用 **SMB 3**，在 macOS 上性能优于 AFP。

3.  **禁止聚光灯索引 (Spotlight)**
    *   **macOS 设置**：在 Mac 的系统设置 > Siri 与聚焦 > 聚焦隐私 > 添加 NAS 的挂载卷。防止 Mac 试图为 NAS 上数百万个音频小文件建立索引，导致卡顿。

## 2. 采样库同步与管理

如何在工作室和家里的电脑上保持采样库一致？

### Synology Drive 单向同步

*   **场景**：通常你在工作室下载并整理好采样库，希望家里的电脑自动更新。
*   **设置**：
    1.  在工作室电脑（主库）设置 Drive **双向同步** 或 **单向上传**。
    2.  在家里的电脑设置 Drive **单向下载**（只读）。
    *   **注意**：采样库文件数量巨大，初次同步会很慢，建议在局域网内完成初次同步。

## 3. Demo 试听与交付系统

别再用网盘发 Demo 了，搭建你自己的私有流媒体服务器。

### 方案 A：Audio Station (简单直接)

1.  将导出的 mp3/wav 放入 `/music/Demo_Showcase` 文件夹。
2.  在 Audio Station 中为该文件夹创建“公开共享链接”。
3.  **优势**：客户打开链接就能在线播放，无需下载，支持手机端。
4.  **劣势**：界面比较传统。

### 方案 B：Navidrome / Plex (专业美观)

*   **Docker 部署 Navidrome**：轻量级，支持无损格式，Web 界面现代。
*   **Docker 部署 Plex**：界面华丽，适合展示成品专辑。
*   **体验**：给客户创建一个账号，他们登录后不仅能听 Demo，还能看到封面、歌词等元数据。

## 4. 自动化备份策略

音乐工程文件一旦损坏，灵感无法重来。

### 3-2-1 备份法则实施

1.  **工作区 (SSD)**：电脑本地或 NAS 的 SSD 卷（第一份数据）。
2.  **快照 (Snapshot)**：NAS 上开启每小时快照（第二份数据，防误删）。
3.  **异地 (Hyper Backup)**：每天凌晨自动备份到公有云（OneDrive/Google Drive）或家里的另一台 NAS（第三份数据，异地）。

### USB Copy 快速冷备

*   买一个大容量移动硬盘插在 NAS 后面。
*   设置 **USB Copy** 任务：当你插入硬盘时，自动将 `01_Projects` 增量备份到硬盘中，备份完成后自动弹出并蜂鸣提醒。
*   **用途**：定期拔下来锁进保险柜，物理隔离。

## 5. 协同创作

### Stem 分轨共享

当需要把分轨发给混音师时：

1.  在 DAW 中导出所有 Stems。
2.  在 File Station 中选中 Stems 文件夹 > **创建文件请求**。
3.  或者直接生成**共享链接**发给混音师。
4.  如果混音师也用群晖，可以使用 **Synology Drive ShareSync** 实现两个 NAS 之间的文件夹实时同步，就像在同一个局域网工作一样。

## 6. 推荐套件与硬件清单

### 必备套件
*   **Audio Station**：基础试听服务。
*   **USB Copy**：物理冷备神器。
*   **Cloud Sync**：将工程同步到百度网盘/Dropbox 作为兜底。
*   **Docker (Container Manager)**：运行 Navidrome 等高级流媒体服务。

### 硬件建议
*   **网络**：10GbE 网卡（必须）。
*   **存储**：NVMe SSD 缓存或 SSD 存储池（强烈建议）。
*   **内存**：大内存有助于文件系统缓存，加速音频小文件读取。
