# Synology Photos 深度优化与迁移

除了基础的备份和浏览，Synology Photos 还有很多鲜为人知的深度玩法，特别是对于从 Google Photos 迁移过来的用户。

## 1. 人脸识别补丁 (Face Recognition Patch)

**痛点**：如果你使用的是“黑群晖”或者某些不带核显的白群晖型号（如 AMD CPU 的机型），Photos 的人脸识别功能可能无法启用。

**解决方案**：
有一个开源项目 `Synology-Photos-Face-Patch` 可以强制开启人脸识别。
*   **原理**：替换系统库文件，绕过 GPU 检测。
*   **风险**：系统更新后可能失效，需重新打补丁。
*   **操作**：通过 SSH 运行脚本（GitHub 搜索该项目，这里不提供直接脚本以防版本过时）。
*   **效果**：让 CPU 代替 GPU 进行 AI 运算，虽然慢一点，但睡一觉通常能跑完几万张照片。

## 2. 完美迁移 Google Photos

**痛点**：Google Takeout 导出的照片和 JSON 元数据是分离的，直接存入 NAS 会导致拍摄日期变成“上传日期”，时间线全乱。

**工具推荐**：`google-photos-takeout-helper`
1.  从 Google Takeout 下载所有照片数据。
2.  在电脑上运行该工具（Python 脚本）。
3.  **功能**：它会自动读取 JSON 中的日期、GPS 信息，并将其写入 JPG/HEIC 文件的 Exif 信息中。
4.  将处理后的照片上传到 NAS，时间线完美恢复。

## 3. 修复 Live Photos 播放问题

**痛点**：iPhone 的 Live Photo 是一张 HEIC 图片 + 一个 MOV 视频。如果通过非官方方式（如 SMB）上传，Photos 可能把它们当成两个独立文件，而不是一个动态照片。

**技巧**：
*   **文件名必须一致**：例如 `IMG_1234.HEIC` 和 `IMG_1234.MOV`。
*   **重新索引**：如果已经上传但未合并，尝试将它们移出再移入，或点击“重建索引”。
*   **使用官方 App**：最稳妥的方式是使用 Synology Photos 手机 App 备份，它会自动处理这种关联。

## 4. 批量修改错误时间 (ExifTool)

**场景**：老照片扫描件、网上下载的图片没有拍摄日期，导致都在“今天”显示。

**CLI 大招**：在 SSH 中安装 `exiftool`（通过 Entware）。
```bash
# 将当前目录下所有 jpg 文件的拍摄日期改为 1990年1月1日
exiftool -AllDates="1990:01:01 12:00:00" *.jpg
```
修改完后，在 Photos 中选择“重建索引”，照片就会乖乖回到 1990 年。

## 5. 共享空间权限 ACL 详解

**痛点**：只想给家人看“宝宝”相册，不想让他看“工作”相册。

**默认机制**：共享空间默认是“全员可见”或“仅管理员可见”。
**精细化操作**：
1.  进入 **Photos 设置** > **共享空间** > **设置访问权限**。
2.  将用户设为 **自定义**。
3.  进入 **相册** 界面，选中文件夹，右键 **设置文件夹权限**。
4.  这里可以像 Windows 文件夹一样，为每个子文件夹单独赋予 User A (查看)、User B (上传) 权限。
5.  **注意**：父文件夹必须至少有“查看”权限，子文件夹的权限才能生效。

## 6. 忽略特定文件 (.plexignore)

虽然 Photos 没有 `.plexignore` 这种文件，但我们可以通过 **控制面板** > **索引服务** > **配置索引文件夹** 来排除。
*   如果你在 `/photo` 下放了大量非照片素材（如游戏贴图），务必将其排除在索引之外，否则你的“最近添加”会被垃圾图片淹没。

## 7. 活用“人物”合并与隐藏

**技巧**：
*   **合并**：AI 经常把同一个人识别成两个（比如戴眼镜和不戴眼镜）。在“人物”相册中，选中两个头像，点击“合并”。
*   **隐藏**：路人甲也被识别出来了？选中头像，点击“隐藏/移除”，它就不会出现在人物列表中了。
*   **封面**：觉得某个人物的封面太丑？进入该人物相册，选中一张美图，右键“设为封面”。

## 8. HEIC/HEVC 解码问题

**痛点**：在网页版看 iPhone 照片黑屏或提示无法播放。

**原因**：浏览器原生不支持 HEVC。
**解法**：
1.  在 NAS 上安装 **Advanced Media Extensions (AME)** 套件。
2.  登录 Synology 账户激活（必须）。
3.  这是群晖官方的解码包，安装后 Photos 会在后台生成兼容的 JPEG 预览图供网页端流畅查看。

## 9. 这里的“相册”不是“文件夹”

**概念区分**：
*   **文件夹模式**：对应物理文件结构 (`/photo/2023/旅游`)。
*   **相册模式**：虚拟的播放列表。
*   **技巧**：你可以把 `/photo/2021`、`/photo/2022` 里的精选照片添加到一个名为“我的精选”的**相册**中。这不会移动原始文件，删除相册也不会删除文件。

## 10. 手机端释放空间

**终极用途**：买 NAS 就是为了给手机省空间。
1.  确保手机 App 显示“备份完成”。
2.  点击 App 设置 > **释放空间**。
3.  App 会对比 NAS 上的指纹，确认已备份后，自动删除手机相册里的原图。
4.  **后悔药**：删除后的照片在 iPhone 的“最近删除”里还能保留 30 天。
